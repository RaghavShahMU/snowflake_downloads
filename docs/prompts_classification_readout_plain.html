Prompts classification readout

Last updated: 2026-04-03

Scope: Classification dimensions only (no triggers or tools in the success model). Correlation sections add tools, triggers, and templates for context.

Data: Classified cohort (agents in agent_classifications.csv with success_segment from cohort).

Value meanings: semantic_layer/models/agent_classifications.md.

Maintenance: This page is maintained manually alongside prompts_classification_readout.md. Chart files and leadership_corr_heatmap_map.json live under analysis/output/ from full_feature_readout_analysis.py and related scripts—not from a doc generator.

Executive summary

  • Purpose: Link prompt classifications to success vs non-success (same cohort as the main readout).
  • Prompts-only model: Random Forest, SHAP, logistic regression on classification one-hot features only.
  • Correlations: Agent-level Pearson vs tools/triggers/templates for context.
  • Lead takeaway: Collection-scoped execution and monitor archetype track with success; single-user prompt and creator track the other way—expand below.

Detailed data analysis and methodologies — See Methodology and footnotes.

Key findings

Finding 1 — Execution dataset — Scoped collections = strongest positive signal; reactive user prompts = sharpest negative.
  • collection_scoped — Highest RF (0.174), LR +0.346.
  • single_user_prompt — Strong negative (RF 0.069, LR -0.273).
  • collection_unbounded — RF 0.047, LR +0.228; single_event_data / single_asset_from_user — RF 0.031/0.027, LR -0.068/-0.258.
