# Semantic model: super_agent_run

**Last updated:** 2025-03-10  
**Source:** `data/super_agent_run.csv`  
**Grain:** One row per agent run (triggered execution). One agent can have many runs.  
**Row count (reference):** 3,205,749  
**Join to super_agents:** `AGENT_ID` → `super_agents.AGENT_ID`

All triggered (executed) agent runs in ClickUp: run identity, trigger context, timing, LLM usage, tool calls, and cost/credits.

---

## Relationship

- **super_agent_run** has many runs per agent; **super_agents** has one row per agent.
- Join key: `super_agent_run.AGENT_ID = super_agents.AGENT_ID`.
- Use this table for run-level metrics (count of runs, duration, cost); use super_agents for agent-level attributes (status, features, workspace).

## Suggested dimensions (for grouping/filtering)

- `AGENT_ID` — join to super_agents, group by agent.
- `TRIGGER_SOURCE` — automation, scheduled, dm, introduction, etc.
- `ENTITLEMENT_NAME` — super_agent_usages vs custom_ai_agent_usages.
- `AGENT_MODEL` — model used.
- Time dimensions from `RUN_STARTED_AT` (year, quarter, month, week, day, hour).
- `WORKSPACE_ID`, `RUN_BY_USER_ID` — for workspace- or user-level slicing.

## Suggested measures (aggregations)

- **Run count** — total runs, runs per agent, runs per workspace, runs per trigger source.
- **Distinct agents** — count distinct AGENT_ID (agents that had at least one run).
- **Duration** — sum/avg of RUN_DURATION_MS (cast to numeric).
- **LLM usage** — sum/avg of TOTAL_LLM_CALL_COUNT, CLICKUP_TOOL_CALL_COUNT, AI_ACTIONS_CONSUMED.
- **Cost** — sum of CUSTOMER_BILLED_COST, EXTERNAL_LLM_CALL_COST, etc. (cast to numeric).
